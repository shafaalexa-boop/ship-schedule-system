# 🚀 Deployment Guide

## Option 1: GitHub Pages (FREE & Recommended)

### Step 1: Enable GitHub Pages
1. Go to your repository: https://github.com/shafaalexa-boop/ship-schedule-system
2. Click **Settings** tab
3. Scroll to **Pages** section (on the left sidebar)
4. Under "Source", select:
   - Branch: `main`
   - Folder: `/ (root)`
5. Click **Save**

### Step 2: Wait for Deployment
- GitHub will automatically build your site
- You'll see a green checkmark when ready
- Your site will be live at:
  - **Jadwal Kapal**: `https://shafaalexa-boop.github.io/ship-schedule-system/ship-schedule/`
  - **Digital Clock**: `https://shafaalexa-boop.github.io/ship-schedule-system/digital-clock/`

### Step 3: Share Your Links
- Direct Links:
  - Ship Schedule: `https://shafaalexa-boop.github.io/ship-schedule-system/ship-schedule/index.html`
  - Digital Clock: `https://shafaalexa-boop.github.io/ship-schedule-system/digital-clock/index.html`

## Option 2: Netlify (Alternative)

### Step 1: Connect to Netlify
1. Go to [netlify.com](https://netlify.com)
2. Click "Add new site" → "Import an existing project"
3. Select GitHub and authorize
4. Choose your repository: `ship-schedule-system`
5. Build settings:
   - Build command: (leave blank)
   - Publish directory: (leave blank)
6. Click "Deploy site"

### Step 2: Your Site is Live
- You'll get a Netlify URL like: `https://your-site-name.netlify.app`
- Both apps will be accessible from this domain

---

## Option 3: Vercel

### Step 1: Deploy to Vercel
1. Go to [vercel.com](https://vercel.com)
2. Click "New Project" → "Import Git Repository"
3. Select your GitHub repo
4. Click "Deploy"

### Step 2: Your Site is Live
- Vercel provides a custom domain automatically
- URLs will be automatically optimized

## 🐛 Troubleshooting

### Pages not showing?
- Wait 2-3 minutes for GitHub to build
- Clear browser cache (Ctrl+Shift+Delete)
- Try incognito/private mode

### Custom domain?
- Go to Settings → Pages → Custom domain
- Enter your domain name
- Update DNS settings (follow GitHub's guide)

### Issues with files?
- Check file paths are lowercase
- Ensure all files are in correct directories
- Verify index.html exists in each folder
