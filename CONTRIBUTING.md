# Contributing

Thank you for your interest in contributing to Ship Schedule System!

## How to Contribute

### Reporting Bugs
1. Check existing issues first
2. Create a new issue with:
   - Clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Browser/OS information

### Suggesting Features
1. Create an issue with title: `[Feature Request]`
2. Describe the feature and why it would be useful
3. Provide examples if possible

### Code Contributions
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make changes following code style
4. Test thoroughly
5. Commit with clear messages: `git commit -m 'feat: description'`
6. Push to your fork
7. Create a Pull Request

## Code Style

- Use 4 spaces for indentation
- Use const/let (not var)
- Add comments for complex logic
- Keep functions focused and small
- Use meaningful variable names

## Testing

- Test on multiple browsers (Chrome, Firefox, Safari, Edge)
- Test on mobile devices
- Verify accessibility (keyboard navigation, screen readers)
- Check dark mode functionality
- Test all search/filter features
